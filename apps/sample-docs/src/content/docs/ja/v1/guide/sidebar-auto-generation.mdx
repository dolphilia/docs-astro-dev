# サイドバーの自動生成

このガイドでは、コンテンツ構造に基づいてサイドバーナビゲーションを自動的に作成するサイドバー自動生成機能の使い方を説明します。

## 概要

サイドバー自動生成機能では、以下のことが可能です：

1. コンテンツファイルに基づいてサイドバーナビゲーションを自動生成
2. フロントマターを使用してカテゴリとその順序を定義
3. カテゴリ内の項目の順序を制御
4. 設定ファイルを手動で更新せずにサイドバー表示をカスタマイズ

## 自動生成の有効化

サイドバー自動生成を有効にするには、`docs.config.ts`ファイルで`useAutoSidebar`オプションを`true`に設定します：

```typescript
// apps/sample-docs/src/config/docs.config.ts
const docsConfig = {
  // ... その他のオプション
  
  // サイドバー自動生成を有効化
  useAutoSidebar: true,
  
  // ... その他のオプション
};
```

## フロントマターの設定

サイドバーでのコンテンツの表示方法を制御するには、以下のフロントマタープロパティを使用します：

```yaml
---
title: "ドキュメントのタイトル"
description: "ドキュメントの説明"
category: "guide"  # カテゴリ名（省略可、デフォルトはパスベースのカテゴリ）
categoryOrder: 1   # サイドバーでのカテゴリの順序（省略可）
order: 2           # カテゴリ内での順序（省略可）
---
```

### プロパティ

| プロパティ | 型 | 説明 |
|----------|------|-------------|
| `title` | string | ドキュメントのタイトル（必須） |
| `category` | string | サイドバーでのグループ化に使用するカテゴリ名（省略可） |
| `categoryOrder` | number | サイドバーでのカテゴリの順序（数値が小さいほど先に表示） |
| `order` | number | カテゴリ内でのドキュメントの順序（数値が小さいほど先に表示） |

## 動作の仕組み

サイドバー生成プロセスは以下の手順で行われます：

1. 現在の言語とバージョンのすべてのコンテンツファイルを収集
2. カテゴリ（フロントマターまたはファイルパスから）でグループ化
3. `categoryOrder`に基づいてカテゴリを並べ替え
4. `order`に基づいて各カテゴリ内のドキュメントを並べ替え
5. 適切なタイトルとリンクを持つサイドバー構造を生成

## パスベースのカテゴリ

フロントマターで`category`を指定しない場合、システムはパスベースのカテゴリを使用します。例えば：

- `src/content/docs/ja/v1/guide/getting-started.mdx`のファイルは「guide」としてカテゴリ分けされます
- `src/content/docs/ja/v1/api/reference.mdx`のファイルは「api」としてカテゴリ分けされます

## カテゴリタイトル

カテゴリタイトルは以下の順序で決定されます：

1. `docs.{category}`に一致する翻訳キー（例：`docs.guide` → 「ガイド」）
2. 翻訳が見つからない場合、カテゴリ名の最初の文字を大文字にします（例：「guide」→ 「Guide」）

## 例

以下はドキュメントのフロントマターの完全な例です：

```yaml
---
title: "入門"
description: "プラットフォームの使い方を学ぶ"
category: "guide"
categoryOrder: 1
order: 1
pubDate: 2025-01-01
updatedDate: 2025-01-01
author: "ドキュメントチーム"
next:
  text: "インストール"
  link: "/ja/v1/guide/installation"
---
```

これにより、ドキュメントは「ガイド」カテゴリ（サイドバーで最初に表示される）に配置され、そのカテゴリ内の最初の項目になります。
